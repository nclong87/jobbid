Nguyên nhân và cách phòng tránh local attack
Đăng bởi: Admin | Ngày: 26/09/2010 | Lần đọc: 1232
Nguyên nhân của việc website bị tấn công với phương pháp local attack, do những sai lầm trong việc quản trị website. Hầu hết những sai lầm đó đều xuất phát từ việc chmod các file và thư mục trên hosting sai. Dẫn đến việc hacker có thể dễ dàng đọc được nội dung mã nguồn website, quan trọng hơn là file chứa thông tin kết nối cơ sở dữ liệu.

Vì vậy để làm thế nào tránh được việc bị hacker tấn công:

Cách CHMOD để tránh hacker tấn công bằng phương pháp local attack:

Đối với thư mục quan trọng: chmod 701

Đối với thư mục bình thường: chmod 755

Đối với file quan trọng như những file kết nối cơ sở dữ liệu: chmod 600 hoặc 400

Đối với những file bình thường: chmod 644

Nên Zend code các file quan trọng để bảo mật cho mã nguồn của website riêng.
